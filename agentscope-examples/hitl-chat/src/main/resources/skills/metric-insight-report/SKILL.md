### 核心执行协议 (Core Execution Protocol)

本 Skill 的所有操作必须严格依赖工具调用（Tool Calling）。严禁在工具返回结果前自行推断数据或生成结论。

#### 1. 第一步：Query 可行性分析 (Tool Call: `analyze_query_feasibility`)
- **操作强制性**：接收到请求后，必须首先调用此工具。
- **核心逻辑**：该工具负责对用户提出的指标查询进行语法、权限及数据完整性校验。
- **分支逻辑 (关键)**：
  - **IF** 工具返回 `status: "unfeasible"` (或表示不可分析的状态)：**必须立即停止**后续所有操作。直接向用户反馈工具提供的错误原因（如：指标未定义、数据缺失等）。
  - **IF** 工具返回 `status: "feasible"` (可行)：提取工具输出的参数化查询指令或中间洞察数据，进入下一步。

#### 2. 第二步：生成洞察报告 (Tool Call: `generate_insight_report`)
- **执行前提**：仅在第一步校验通过（Feasible）的情况下触发。
- **操作指令**：调用此工具将第一步确认的可行查询或初步结果转化为最终报告。
- **核心逻辑**：工具负责执行底层数据聚合（如 ClickHouse 查询）并将结果格式化为业务可读的 Markdown 报告。
- **最终交付**：直接透传并展示此工具生成的报告内容。

### 约束与规范 (Constraints)
- **零干扰原则**：若第一步判定为不可分析，严禁调用第二步工具，亦不得尝试自行编写伪数据报告。
- **数据一致性**：报告中的所有指标逻辑必须与第一步校验通过的逻辑保持绝对一致。
