# JSON技能创建器

本技能提供创建有效技能的指南，输出为JSON格式直接返回给调用方。

## 关于技能

技能是模块化、自包含的包，通过提供专业知识、工作流程和工具来扩展智能体的能力。可以将它们视为特定领域或任务的"入门指南"——它们将通用智能体转变为具备程序性知识的专用智能体。

### 技能提供的内容

1. **专用工作流程** - 特定领域的多步骤流程
2. **工具集成** - 处理特定文件格式或 API 的说明
3. **领域专业知识** - 公司特定知识、数据库模式、业务逻辑
4. **使用指导** - 详细的使用说明和示例

## 核心原则

### 简洁是关键

上下文窗口是一种公共资源。只添加智能体没有的上下文。挑战每条信息："智能体真的需要这个解释吗？"和"这段内容是否值得其 token 成本？"

优先使用简洁示例而非冗长解释。

### 设定适当的自由度

将具体程度与任务的脆弱性和可变性相匹配：

- **高自由度（文本指令）**：当多种方法有效、决策取决于上下文时使用
- **中等自由度（伪代码或带参数的脚本）**：当存在首选模式、某些变化可接受时使用
- **低自由度（特定脚本，少量参数）**：当操作脆弱且易出错、一致性至关重要时使用

### 技能的结构

每个技能由必需的 SKILL.md 文件和可选的捆绑资源组成：

```
skill-name/
├── SKILL.md（必需）
│   ├── YAML 前置元数据（必需）
│   │   ├── name:（必需）
│   │   └── description:（必需）
│   └── Markdown 指令（必需）
└── 捆绑资源（可选）
    ├── scripts/          - 可执行代码（Python/Bash 等）
    ├── references/       - 按需加载到上下文中的文档
    └── assets/           - 输出中使用的文件（模板、图标、字体等）
```

## 技能创建流程

技能创建涉及以下步骤：

1. 用具体示例理解技能
2. 规划可复用技能内容
3. 生成JSON格式的技能定义
4. 根据实际使用迭代

### 步骤 1：用具体示例理解技能

要创建有效的技能，需清晰理解技能将如何使用的具体示例。

例如，在构建 image-editor 技能时，相关问题包括：

- "image-editor 技能应支持什么功能？编辑、旋转，还有其他吗？"
- "你能给出一些此技能会如何使用的示例吗？"
- "用户会说什么来触发此技能？"

为避免让用户不知所措，避免在一条消息中问太多问题。从最重要的问题开始，按需跟进以提高效果。

当对技能应支持的功能有清晰认识时，结束此步骤。

### 步骤 2：规划可复用技能内容

要将具体示例转化为有效技能，分析每个示例：

1. 考虑如何从头执行示例
2. 确定重复执行这些工作流程时有用的脚本、参考资料和资产

### 步骤 3：生成JSON格式的技能定义

这是本技能的核心输出。直接返回以下JSON结构，**不要写入文件**，直接作为响应内容返回：

```json
{
    "SKILL.md": {
        "nameCn": "技能中文名（必填，简洁明了）",
        "nameEn": "skill-english-name（必填，使用连字符分隔的小写英文）",
        "description": "技能描述（必填，包含技能做什么以及何时使用的具体触发条件）",
        "content": "技能详细内容（必填，使用Markdown格式，包含完整的使用说明、工作流程、示例等）"
    }
}
```

#### JSON字段说明

| 字段 | 说明 | 要求 |
|------|------|------|
| `nameCn` | 技能的中文名称 | 简洁明了，2-10个汉字 |
| `nameEn` | 技能的英文名称 | 连字符分隔的小写英文，如 `data-analyzer` |
| `description` | 技能描述 | 说明技能功能和使用场景/触发条件，100-500字 |
| `content` | 技能详细内容 | Markdown格式，包含概述、使用指南、示例、注意事项等 |

#### content 内容结构建议

SKILL.md 的 `content` 字段应包含以下部分：

```markdown
# {nameCn}

## 概述

简要说明技能的功能和用途。

## 能力

- 能力1
- 能力2
- 能力3

## 使用场景

### 场景1：xxx

具体使用说明...

### 场景2：xxx

具体使用说明...

## 示例

### 示例1：xxx

输入：...
输出：...

## 注意事项

- 注意事项1
- 注意事项2
```

### 编写指南

1. **使用命令式/不定式形式** - 指令应直接、明确
2. **保持简洁** - 优先使用简洁示例而非冗长解释
3. **包含具体触发条件** - 在 description 中明确说明何时使用此技能
4. **提供工作流指导** - 在 content 中说明多步骤流程
5. **使用中文** - 面向中文用户的技能应使用中文编写

### 步骤 4：迭代

根据实际使用反馈改进技能：

1. 在实际任务中使用技能
2. 注意困难或效率问题
3. 确定如何更新技能定义
4. 实现变更并再次测试

## 总结

当用户使用此技能时：

1. 首先通过提问理解用户需要的技能功能
2. 分析并规划技能的组成部分
3. **直接返回JSON格式的技能定义**，不要写入文件
4. 确保JSON结构完整，包含 nameCn、nameEn、description 和 content 四个字段
