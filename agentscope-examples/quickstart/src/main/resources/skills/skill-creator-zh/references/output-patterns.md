# 输出模式

当技能需要产生一致、高质量的输出时使用这些模式。

## 模板模式

为输出格式提供模板。将严格程度与需求匹配。

**对于严格要求（如 API 响应或数据格式）：**

```markdown
## 报告结构

始终使用此精确模板结构：

# [分析标题]

## 执行摘要
[关键发现的一段式概览]

## 关键发现
- 发现 1 及支持数据
- 发现 2 及支持数据
- 发现 3 及支持数据

## 建议
1. 具体可执行的建议
2. 具体可执行的建议
```

**对于灵活指导（当适应有用时）：**

```markdown
## 报告结构

这是一个合理的默认格式，但请运用最佳判断：

# [分析标题]

## 执行摘要
[概览]

## 关键发现
[根据发现内容调整章节]

## 建议
[针对特定上下文定制]

根据具体分析类型按需调整章节。
```

## 示例模式

对于输出质量取决于看到示例的技能，提供输入/输出对：

```markdown
## 提交消息格式

按以下示例生成提交消息：

**示例 1：**
输入：添加了使用 JWT token 的用户认证
输出：
```
feat(auth): 实现基于 JWT 的认证

添加登录端点和 token 验证中间件
```

**示例 2：**
输入：修复了报告中日期显示不正确的 bug
输出：
```
fix(reports): 修正时区转换中的日期格式

在报告生成中始终使用 UTC 时间戳
```

遵循此风格：类型(范围): 简短描述，然后详细解释。
```

示例帮助 Claude 比描述更清晰地理解所需风格和细节程度。
