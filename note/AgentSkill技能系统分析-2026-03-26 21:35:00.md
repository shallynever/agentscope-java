# AgentSkill 技能系统分析

> 分析时间：2026-03-26 21:35:00

---

## 1、概述（Overview）

**分析对象：** `io.agentscope.core.skill.AgentSkill` 及相关类

**核心目标：** 理解技能系统的渐进式披露机制、工具绑定方式、Hook 集成原理。

---

## 2、原始问题（保真）

1. "好像在Agent执行`doCall`方法的流程中没有发现SKILL的相关逻辑呢"
2. "如何给AgentSkill绑定专属工具"
3. "这样ReActAgent是可以不用AgentSkill，那什么样的场景下需要使用AgentSkill"
4. "agentscope-examples 目录下那些项目使用了 AgentSkill"

---

## 3、分析过程（核心：推理链）

### Step 1：寻找 Skill 集成位置

**当时理解：** 应该在 doCall() 中能看到 Skill 代码

**依据：** 直接阅读 `doCall()` 方法

**是否正确：** ❌ 理解错误

### Step 2：深入分析 Skill 集成机制

**当时理解：** Skill 通过某种机制注入，需要找到集成点

**依据：** 搜索 `skillBox` 关键词，找到 `configureSkillBox()` 方法

**是否正确：** ✅ 正确

### Step 3：关键突破 - 理解渐进式披露

**当时理解：** Skill 内容一次性加载到 prompt

**依据：** 阅读 `SkillHook` 和 `SkillBox`

**是否正确：** ❌ 理解错误

**修正理解：**
- 初始只显示技能描述（~50 tokens）
- 使用时才加载完整内容（~1000 tokens）
- 工具绑定后，只有技能激活才可用

**当时理解：** 工具绑定到 Skill 后立即可用

## 6、核心流程

### 6.1 渐进式披露流程

```
初始状态
    │
    ├── System Prompt 包含技能描述
    │   ┌─────────────────────────────────┐
    │   │ ## Available Skills             │
    │   │ <skill>                         │
    │   │   <name>data_analysis</name>    │
    │   │   <description>数据分析...</description> │
    │   │   <skill-id>data_analysis_xxx</skill-id> │
    │   │ </skill>                        │
    │   └─────────────────────────────────┘
    │   (约 50 tokens)
    │
    ▼

    │
    ▼

    │
    ▼

    │
    ▼
返回完整技能内容
    │   ┌─────────────────────────────────┐
    │   │ # 数据分析技能                   │
    │   │ ## 分析流程                      │
    │   │ 1. 数据清洗                      │
    │   │ 2. 指标计算                      │
    │   │ ...                              │
    │   └─────────────────────────────────┘
    │   (约 1000 tokens)
    │
    ▼

```


## 7、关键代码片段

### 7.1 技能注册

```java
// SkillBox.java
public void apply() {
    skillBox.registerSkill(skill);

    if (有工具) {
        String skillToolGroup = skill.getSkillId() + "_skill_tools";
        toolkit.createToolGroup(skillToolGroup, skillToolGroup, false);  // 默认不激活

        toolkit.registration()
                .group(skillToolGroup)
                .agentTool(agentTool)
                .apply();
    }
}
```

### 7.2 Hook 注入技能提示

```java
// SkillHook.java
public <T extends HookEvent> Mono<T> onEvent(T event) {
    if (event instanceof PreReasoningEvent preReasoningEvent) {
        String skillPrompt = skillBox.getSkillPrompt();
        // 合并到 SYSTEM 消息
        Msg mergedMsg = Msg.builder()
            .role(MsgRole.SYSTEM)
            .content(mergedContent)
            .build();
        preReasoningEvent.setInputMessages(newMessages);
    }
    return Mono.just(event);
}
```

### 7.3 技能激活

```java
// SkillToolFactory
private AgentSkill validatedActiveSkill(String skillId) {
    AgentSkill skill = skillRegistry.getSkill(skillId);
    skillRegistry.setSkillActive(skillId, true);

    String toolsGroupName = skillRegistry.getRegisteredSkill(skillId).getToolsGroupName();
    toolkit.updateToolGroups(List.of(toolsGroupName), true);  // 激活工具组

    return skill;
}
```

## 9、关键机制 / 设计点

### 9.1 渐进式披露

| 阶段 | 内容 | Token 消耗 |
|------|------|-----------|
| 初始 | 技能描述 | ~50 |
| 加载后 | 完整技能内容 | ~1000 |

### 9.2 工具组隔离

| 机制 | 说明 |
|------|------|
| 命名规则 | `{skillId}_skill_tools` |
| 默认状态 | `active = false` |
| 激活时机 | 技能加载时 |

### 9.3 Hook 集成

| Hook | 作用 |
|------|------|
| SkillHook | 拦截 PreReasoningEvent，注入技能提示 |
| priority=55 | 高优先级，在其他 Hook 之前执行 |

---

## 10、关键结论

### 真实执行逻辑

1. **Skill 通过 Hook 集成**：不在 doCall() 中直接调用
2. **渐进式披露**：初始只显示描述，使用时才加载完整内容
3. **工具默认不可用**：绑定到技能的工具组默认不激活
4. **按需激活**：技能加载时自动激活关联工具组

### 与直觉不一致的地方

| 直觉 | 实际 |
|------|------|
| Skill 在 doCall() 中处理 | 通过 Hook 拦截 PreReasoningEvent |
| 工具绑定后立即可用 | 默认不激活，技能加载后才可用 |
| 技能内容一次性加载 | 渐进式披露 |

## 11、错误理解记录

| 误解点 | ❌ 误解 | ✔ 正解 | 原因 |
|--------|--------|--------|------|
| Skill 集成位置 | 应在 doCall() 中 | 通过 Hook 机制集成 | 不了解 AOP 设计模式 |
| 工具激活时机 | 绑定后立即可用 | 技能激活后才可用 | 未发现工具组 active=false |
| 内容加载方式 | 一次性加载 | 渐进式披露 | 未理解 skillPrompt vs skillContent |

## 13、可复用经验

### 设计模式

| 模式 | 应用 |
|------|------|
| **Hook 模式 (AOP)** | 扩展功能不修改核心流程 |
| **渐进式披露** | 按需加载，减少上下文污染 |
| **工厂模式** | SkillToolFactory 创建技能工具 |

### 适用场景判断

```
是否需要 AgentSkill?
         │
         ▼
┌─────────────────────────────┐
│ 技能文档 > 500 字？          │──── 否 ───→ 用 sysPrompt
└─────────────┬───────────────┘
              │ 是
              ▼
┌─────────────────────────────┐
│ 技能数量 > 3 个？            │──── 否 ───→ 用 sysPrompt
└─────────────┬───────────────┘
              │ 是
              ▼
┌─────────────────────────────┐
│ 需要技能专属工具？           │──── 否 ───→ 考虑是否需要
└─────────────┬───────────────┘
              │ 是
              ▼
        使用 AgentSkill
```
