## 3、分析过程（核心：推理链）

### Step 1：初步理解暂停机制

**当时理解：** HITL 通过某种方式暂停 Agent 执行

**依据：** 探索 `PostReasoningEvent.stopAgent()` 方法

**是否正确：** ✅ 正确

---

### Step 2：深入分析暂停点

**当时理解：** 只有一个暂停点

**依据：** 阅读 `PostReasoningEvent` 和 `PostActingEvent`

**是否正确：** ❌ 理解不完整

**修正：** 有三个暂停点：
1. PostReasoningEvent.stopAgent() - 推理后
2. PostActingEvent.stopAgent() - 执行后
3. ToolSuspendException - 工具执行时

---

### Step 3：关键突破 - 恢复机制

**当时理解：** 暂停后需要重新创建 Agent

**依据：** 阅读 `doCall()` 中的 pendingToolUseIds 处理

**是否正确：** ❌ 理解错误

**修正：**
```java
// 暂停后通过 agent.call() 恢复
// 无参数 → 继续执行待处理工具
// 带参数 → 提供自定义结果
```

---

### Step 4：最终确认 - 工具挂起机制

**当时理解：** 工具抛出异常会导致执行失败

**依据：** 阅读 `ToolSuspendException` 和 `ToolExecutor`

## 4、类结构

```
PostReasoningEvent extends ReasoningEvent
    ├── stopRequested: boolean
    ├── gotoReasoningMsgs: List<Msg>
    │
    ├── stopAgent()  → 设置 stopRequested = true
    └── gotoReasoning(msgs)  → 跳回推理阶段

PostActingEvent extends ActingEvent
    ├── stopRequested: boolean
    │
    └── stopAgent()  → 设置 stopRequested = true

ToolSuspendException extends RuntimeException
    ├── reason: String
    │
    └── 用于信号工具需要外部执行

ToolResultBlock
    ├── METADATA_SUSPENDED = "agentscope_suspended"
    │
    ├── isSuspended()  → 检查是否挂起
    └── suspended(toolUse, exception)  → 创建挂起结果

GenerateReason (枚举)
    ├── TOOL_SUSPENDED  → 工具挂起
    ├── REASONING_STOP_REQUESTED  → 推理阶段停止
    └── ACTING_STOP_REQUESTED  → 执行阶段停止
```

## 5、核心调用链

### 5.1 Hook 暂停流程

```
reasoning(iter)
    │
    ▼
notifyPostReasoning(msg)
    │
    ▼
Hook.onEvent(PostReasoningEvent)
    │
    ├── hook 判断需要暂停
    │
    └── event.stopAgent()  → stopRequested = true
    │
    ▼
ReActAgent 检查
    │
    └── if (event.isStopRequested())
            return Mono.just(msg.withGenerateReason(REASONING_STOP_REQUESTED));
```

### 5.2 工具挂起流程

```
acting(iter)
    │
    ▼
toolkit.callTools()
    │
    ▼
ToolExecutor.executeCore()
    │
    ▼
tool.callAsync()
    │
    ├── throw new ToolSuspendException(reason)
    │
    ▼
onErrorResume(ToolSuspendException.class, e -> {
    return Mono.just(ToolResultBlock.suspended(toolCall, e));
})
    │
    ▼
返回 ToolResultBlock (isSuspended = true)
    │
    ▼
ReActAgent 检查
    │
    └── if (pendingPairs.isEmpty() == false)
            return Mono.just(buildSuspendedMsg(pendingPairs));
```

### 5.3 恢复执行流程

```
agent.call()  // 无参数
    │
    ▼
doCall(List<Msg> msgs)
    │
    ├── getPendingToolUseIds() → 发现有待处理工具
    │
    └── acting(0)  → 继续执行

agent.call(toolResultMsg)  // 带参数
    │
    ▼
doCall(List<Msg> msgs)
    │
    ├── validateAndAddToolResults(msgs, pendingIds)
    │       │
    │       └── 验证工具结果与待处理工具匹配
    │
    └── hasPendingToolUse() ? acting(0) : executeIteration(0)
```

## 6、核心流程

### 6.1 三暂停点流程

```
reasoning()
    │
    ├─────────────────────────────────────────┐
    │ [暂停点1] PostReasoningEvent.stopAgent()│
    │ - LLM 已决定调用哪些工具                │
    │ - 用户看到工具名和参数                   │
    │ - 可选择确认/取消                        │
    └─────────────────────────────────────────┘
    │
    ▼
acting()
    │
    ├─────────────────────────────────────────┐
    │ [暂停点2] 工具执行时                     │
    │ - 工具抛出 ToolSuspendException          │
    │ - 等待用户/外部系统提供结果               │
    └─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│ [暂停点3] PostActingEvent.stopAgent()       │
│ - 工具已执行完成                            │
│ - 用户看到执行结果                          │
│ - 可选择继续/停止                           │
└─────────────────────────────────────────────┘
```

### 6.2 确认与恢复流程

```
用户提问
    │
    ▼
LLM 决定调用 delete_file
    │
    ▼
Hook 检测到敏感工具 → stopAgent()
    │
    ▼
返回 GenerateReason.REASONING_STOP_REQUESTED
    │
    ▼
┌─────────────────────────────────────┐
│ 显示: 即将执行 delete_file("xxx")   │
│ 是否继续? (y/n)                     │
└─────────────────────────────────────┘
    │
    ├── y → agent.call()  → 继续执行
    │
    └── n → agent.call(cancelResult)  → 提供取消结果
```

## 8、数据流

### 8.1 暂停状态数据流

```
正常执行:
    ToolUseBlock (id="abc", name="delete_file")
        │
        ▼
    执行工具
        │
        ▼
    ToolResultBlock (id="abc", content="删除成功")

暂停后:
    ToolUseBlock (id="abc", name="delete_file")
        │
        ▼
    返回 Msg 包含 ToolUseBlock
    GenerateReason = REASONING_STOP_REQUESTED
        │
        ▼
    Memory 中保存: ToolUseBlock (无对应 ToolResultBlock)
```

### 8.2 恢复执行数据流

```
用户确认:
    │
    ▼
agent.call()  // 无参数
    │
    ▼
Memory 中找到 pending ToolUseBlock
    │
    ▼
acting(0) → 执行工具
    │
    ▼
ToolResultBlock 添加到 Memory

用户取消:
    │
    ▼
agent.call(cancelResultMsg)
    │
    ▼
cancelResultMsg 包含 ToolResultBlock (id="abc", content="用户取消")
    │
    ▼
添加到 Memory，继续执行
```

## 9、关键机制 / 设计点

### 9.1 三个暂停点对比

| 暂停点 | 触发方式 | 看到内容 | 适用场景 |
|--------|----------|----------|----------|
| PostReasoning | `stopAgent()` | 工具调用意图 | 执行前确认 |
| Tool 执行时 | `ToolSuspendException` | 工具调用参数 | 收集用户输入 |
| PostActing | `stopAgent()` | 工具执行结果 | 执行后检查 |

### 9.2 GenerateReason 标识

| 值 | 含义 |
|---|------|
| `TOOL_SUSPENDED` | 工具被挂起 |
| `REASONING_STOP_REQUESTED` | 推理阶段停止 |
| `ACTING_STOP_REQUESTED` | 执行阶段停止 |

### 9.3 恢复方法

| 方法 | 说明 |
|------|------|
| `agent.call()` | 继续执行待处理工具 |
| `agent.call(resultMsg)` | 提供自定义结果并继续 |

## 10、关键结论

### 真实执行逻辑

1. **Hook 暂停**：通过 `stopAgent()` 设置标志位，Agent 检查后返回
2. **工具挂起**：抛出 `ToolSuspendException`，框架捕获并转为挂起状态
3. **恢复执行**：Memory 保留状态，通过 `call()` 恢复

### 设计优势

| 特性 | 说明 |
|------|------|
| **非侵入式** | 通过 Hook 机制，不修改核心流程 |
| **灵活性** | 支持多种暂停点和恢复方式 |
| **状态保持** | 暂停时 Memory 保留，恢复后继续上下文 |

## 11、错误理解记录

| 误解点 | ❌ 误解 | ✔ 正解 | 原因 |
|--------|--------|--------|------|
| 暂停点数量 | 只有 1 个 | 有 3 个暂停点 | 未全面探索 |
| 恢复方式 | 需要重新创建 Agent | 通过 call() 恢复 | 未理解 Memory 保留机制 |
| 工具挂起 | 会导致执行失败 | 是设计行为，框架捕获处理 | 不了解 ToolSuspendException |

## 13、可复用经验

### 使用场景

| 场景 | 方案 |
|------|------|
| 敏感操作确认 | PostReasoningEvent.stopAgent() |
| 收集用户输入 | ToolSuspendException |
| 外部系统执行 | SchemaOnlyTool + ToolSuspendException |
| 执行结果审查 | PostActingEvent.stopAgent() |

### 代码模式

```java
// 1. 确认 Hook
Hook confirmationHook = new Hook() {
    public <T extends HookEvent> Mono<T> onEvent(T event) {
        if (event instanceof PostReasoningEvent e) {
            if (hasSensitiveTool(e.getReasoningMessage())) {
                e.stopAgent();
            }
        }
        return Mono.just(event);
    }
};

// 2. 用户交互工具
@Tool(description = "询问用户")
public String askUser(String question) {
    throw new ToolSuspendException(question);
}

// 3. 处理暂停
Msg response = agent.call(userMsg).block();
if (response.getGenerateReason() == GenerateReason.REASONING_STOP_REQUESTED) {
    if (userConfirms()) {
        response = agent.call().block();  // 继续
    } else {
        response = agent.call(createCancelResult()).block();  // 取消
    }
}
```
