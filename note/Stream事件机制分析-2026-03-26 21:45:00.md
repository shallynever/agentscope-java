## 6、核心流程

### 6.1 事件生成流程

```
call(msgs) 执行
    │
    ▼
reasoning(0)
    │
    ├── LLM 流式输出
    │       │
    │       └── ReasoningChunkEvent
    │               │
    │               └── StreamingHook.onEvent()
    │                       │
    │                       ├── options.shouldStream(REASONING)? ✓
    │                       ├── options.shouldIncludeReasoningEmission(true)? ✓
    │                       │
    │                       └── emitEvent(REASONING, chunk, isLast=false)
    │
    └── 推理完成
            │
            └── PostReasoningEvent
                    │
                    └── StreamingHook.onEvent()
                            │
                            ├── options.shouldIncludeReasoningEmission(false)? ✓
                            │
                            └── emitEvent(REASONING, msg, isLast=true)
    │
    ▼
acting(0)
    │
    ├── 工具执行中
    │       └── ActingChunkEvent → Event(TOOL_RESULT, chunk, false)
    │
    └── 执行完成
            └── PostActingEvent → Event(TOOL_RESULT, result, true)
    │
    ▼
call() 返回
    │
    └── Event(AGENT_RESULT, finalMsg, true)
```

### 6.2 EventType 判断流程

```
Hook 事件
    │
    ├── instanceof ReasoningChunkEvent?
    │       └── 是 → EventType.REASONING, isLast=false
    │
    ├── instanceof PostReasoningEvent?
    │       └── 是 → EventType.REASONING, isLast=true
    │
    ├── instanceof ActingChunkEvent?
    │       └── 是 → EventType.TOOL_RESULT, isLast=false
    │
    ├── instanceof PostActingEvent?
    │       └── 是 → EventType.TOOL_RESULT, isLast=true
    │
    ├── instanceof SummaryChunkEvent?
    │       └── 是 → EventType.SUMMARY, isLast=false
    │
    ├── instanceof PostSummaryEvent?
    │       └── 是 → EventType.SUMMARY, isLast=true
    │
    └── 其他 → 不生成 Event
```

## 7、关键代码片段

### 7.1 createEventStream 核心逻辑

```java
// AgentBase.java
private Flux<Event> createEventStream(StreamOptions options, Supplier<Mono<Msg>> callSupplier) {
    return Flux.<Event>create(sink -> {
        StreamingHook streamingHook = new StreamingHook(sink, options);
        addHook(streamingHook);

        Mono.defer(() -> callSupplier.get())
            .doFinally(signalType -> hooks.remove(streamingHook))
            .subscribe(
                finalMsg -> {
                    if (options.shouldStream(EventType.AGENT_RESULT)) {
                        sink.next(new Event(EventType.AGENT_RESULT, finalMsg, true));
                    }
                    sink.complete();
                },
                sink::error);
    });
}
```

### 7.2 StreamingHook 事件映射

```java
// StreamingHook.java
public <T extends HookEvent> Mono<T> onEvent(T event) {
    if (event instanceof ReasoningChunkEvent e) {
        if (options.shouldStream(EventType.REASONING)
            && options.shouldIncludeReasoningEmission(true)) {
            Msg msgToEmit = options.isIncremental()
                ? e.getIncrementalChunk()
                : e.getAccumulated();
            emitEvent(EventType.REASONING, msgToEmit, false);
        }
    } else if (event instanceof PostReasoningEvent e) {
        if (options.shouldStream(EventType.REASONING)
            && options.shouldIncludeReasoningEmission(false)) {
            emitEvent(EventType.REASONING, e.getReasoningMessage(), true);
        }
    } else if (event instanceof PostActingEvent e) {
        if (options.shouldStream(EventType.TOOL_RESULT)) {
            emitEvent(EventType.TOOL_RESULT, createToolMessage(e.getToolResult()), true);
        }
    }
    // ... 其他事件
    return Mono.just(event);
}
```

### 7.3 StreamOptions 过滤

```java
// StreamOptions.java
public boolean shouldStream(EventType type) {
    return eventTypes.contains(EventType.ALL) || eventTypes.contains(type);
}

public boolean shouldIncludeReasoningEmission(boolean isChunk) {
    return isChunk ? includeReasoningChunk : includeReasoningResult;
}
```

### 7.4 emitEvent

```java
// StreamingHook.java
private void emitEvent(EventType type, Msg msg, boolean isLast) {
    Event event = new Event(type, processedMsg, isLast);
    sink.next(event);
}
```

## 9、关键机制 / 设计点

### 9.1 Event 对比表

| 场景 | Hook 事件 | EventType | isLast | Message Role |
|------|-----------|-----------|--------|--------------|
| LLM 流式块 | ReasoningChunkEvent | REASONING | false | ASSISTANT |
| LLM 推理完成 | PostReasoningEvent | REASONING | true | ASSISTANT |
| 工具流式块 | ActingChunkEvent | TOOL_RESULT | false | TOOL |
| 工具完成 | PostActingEvent | TOOL_RESULT | true | TOOL |
| 总结流式块 | SummaryChunkEvent | SUMMARY | false | ASSISTANT |
| 总结完成 | PostSummaryEvent | SUMMARY | true | ASSISTANT |
| 最终结果 | call() 返回值 | AGENT_RESULT | true | ASSISTANT |

### 9.2 isLast 含义

| 值 | 含义 | 用途 |
|---|------|------|
| `true` | 完整消息或最终块 | 可持久化、显示为最终结果 |
| `false` | 中间流式块 | 实时 UI 更新 |

### 9.3 incremental 模式

| 模式 | 说明 |
|------|------|
| `incremental=true` | 只返回新增内容 |
| `incremental=false` | 返回累积内容 |

## 10、关键结论

### 真实执行逻辑

1. **Event 通过 Hook 生成**：`StreamingHook` 拦截 Hook 事件转换为 Event
2. **EventType 由 Hook 事件类型决定**：不同 Hook 事件映射不同 EventType
3. **isLast 标识完整性**：`false`=中间块，`true`=完整消息
4. **StreamOptions 控制过滤**：可配置接收哪些事件类型

### 设计优势

| 特性 | 说明 |
|------|------|
| **非侵入式** | 通过 Hook 机制，不修改核心流程 |
| **灵活过滤** | StreamOptions 控制接收哪些事件 |
| **流式支持** | 支持实时 UI 更新 |

## 11、错误理解记录

| 误解点 | ❌ 误解 | ✔ 正解 | 原因 |
|--------|--------|--------|------|
| Event 生成方式 | Agent 直接生成 | 通过 Hook 拦截转换 | 未理解 Hook 机制 |
| EventType 判断 | 复杂判断逻辑 | 直接根据 Hook 事件类型映射 | 未阅读 StreamingHook |
| isLast 含义 | 标识最终事件 | 标识消息完整性 | 概念混淆 |

## 13、可复用经验

### 使用模式

```java
// 1. 只接收推理事件
StreamOptions options = StreamOptions.builder()
    .eventTypes(EventType.REASONING)
    .includeReasoningChunk(true)
    .incremental(true)
    .build();

// 2. 只接收工具结果
StreamOptions options = StreamOptions.builder()
    .eventTypes(EventType.TOOL_RESULT)
    .includeActingChunk(false)  // 不要流式块
    .build();

// 3. 只接收最终结果
StreamOptions options = StreamOptions.builder()
    .eventTypes(EventType.AGENT_RESULT)
    .build();

// 4. 订阅处理
agent.stream(userMsg, options)
    .subscribe(event -> {
        switch (event.getType()) {
            case REASONING -> {
                if (event.isLast()) {
                    // 完整消息
                } else {
                    // 流式块
                }
            }
            case TOOL_RESULT -> {
                // 工具结果
            }
            case AGENT_RESULT -> {
                // 最终结果
            }
        }
    });
```
